## 实现完成的四个任务

### Problem 1: 轮询监听 (Listen to the Round Robin) - ✅
**核心机制**: PING/REPLY/ANNOUNCE 网络发现 + TOKEN 令牌传递

**主要代码**:
- `do_discovery()` - 发起发现，500ms 等待回复
- `discovery_timeout()` - 收集 active_nodes，计算 rank，发送第一个 TOKEN
- `handle_discovery_ping/reply/announce()` - 处理发现消息
- `send_token()` 和 `handle_token()` - 令牌传递机制
- `play_one_note()` / `finish_note()` - 替代原有独立的 play_note/stop_note
- `periodic_print()` - 每 10 秒打印 tempo（指挥）或 MUTED（音乐家）

**协议细节**:
- Node ID: 0-14（小组编号）
- Rank: 在 active_nodes 中的位置（0-based）
- 轮询规则: 音符 N 由 rank = N % active_count 的板子演奏
- TOKEN 格式: msgId=5, buff[0..1]=uint16 note_index

### Problem 2: 拥抱你的新指挥 (Embrace Your New Conductor) - ✅
**核心机制**: CONDUCTOR_ANNOUNCE（声称型）+ Rank 优先级

**主要代码**:
- `claim_conductorship()` - 发送声称消息（length=1）
- `handle_conductor_announce()` - 处理声称；低 nodeId 获胜
- `become_conductor()` / `become_musician()` - 角色转换

**行为**:
- 演奏者输入 `claim` 或 5 秒长按 USER 按钮
- 冲突解决: nodeId 越小优先级越高
- 输出消息: 新指挥"Claimed Conductorship"，旧指挥"Conductorship Void"
- 旋律连续性: 继续使用之前的 tempo/key，不中断

### Problem 3: 沉默之声 (The Sound of Silence) - ✅
**核心机制**: 心跳监测 + 阶梯看门狗 + 被动备份

**主要代码**:
- `enter_silent_failure()` / `leave_silent_failure()` - 进入/离开故障状态
- `send_note_hb()` - 10ms 心跳（演奏中发送）
- `watchdog_fire()` - 200ms/400ms/600ms 看门狗触发
- `f2_auto_exit()` - F2 模式自动恢复（5-10s）
- `handle_heartbeat()` - 接收并处理心跳

**故障模式**:
- **F1**: 手动切换（`f1` 命令或 2 秒长按 USER）
- **F2**: 自动恢复（`f2` 命令，5-10s 后自动离开）
- **F3**: CAN 断线检测（可选，需 __CAN_TxAck）

**阶梯看门狗**:
- Rank 0: 200ms
- Rank 1: 400ms  
- Rank 2: 600ms
- 设计原理: 最低rank 的活跃板最先检测到故障，自动接管

**并发故障处理**: 
- 支持最多 2 个同时故障
- 恢复时间 ≤ 1s（每个故障单独计算）
- 故障板离线后，其他板自动推进旋律

### Problem 4: 让指挥者静默 (Silence that Conductor) - ✅
**核心机制**: 指挥者心跳 + 指挥者看门狗 + 自动选举

**主要代码**:
- `send_cond_hb()` - 100ms 指挥者心跳（非演奏时）
- `conductor_wd_fire()` - 指挥者看门狗触发（同阶梯）
- `become_conductor()` with reason=COND_REASON_AUTO - 自动选举
- 旧指挥恢复逻辑: 变为演奏者，打印"I Now Join As A Musician"

**自动选举**:
- 最低 rank 的活跃板自动成为新指挥
- 发送完整 CONDUCTOR_ANNOUNCE
- 新指挥继续旋律（从 last_token_index）

**死网络恢复**:
- 所有板都静默故障 → 无旋律
- 第一块板恢复 → 自动成为指挥、发起发现、从头播放

**指挥者心跳**:
- 演奏中: 10ms 心跳（由 play_one_note 发起）
- 非演奏时: 100ms 心跳（由 become_conductor 发起）

## 关键设计决策

1. **令牌传递优于独立定时器**: 避免时钟漂移，确保同步
2. **阶梯看门狗**: rank0 最先检测，避免并发竞争
3. **会话 (session) 机制**: 用于失效旧的异步回调
4. **分离的心跳**: 10ms (演奏) vs 100ms (指挥) 两套独立计时器
5. **被动备份**: 只有检测到故障才接管（无抢占）
6. **自动选举**: 指挥故障时最低 rank 的板自动升级

## 已知限制

1. 最多 3-4 块板（active_nodes 大小 16，nodeId 0-14）
2. F3 检测需要 __CAN_TxAck 编译选项（用于 1 秒内恢复）
3. 无 USER 按钮实现（代码支持结构但需硬件中断）
4. 单板测试需要 CAN 环回（已在代码中支持）
