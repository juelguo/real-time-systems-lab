# HWA1 协作部分：四个任务实现计划

**✓ 实现完成** – 2026-04-15
- 编译成功（build/real-time-systems-lab.s19）
- 所有四个任务的代码已写入 App.h 和 App.c

## 背景

本项目是基于 TinyTimber RTOS 的 CAN 总线分布式旋律播放器（MD407/STM32F4 硬件）。四个协作任务（P1–P4）需在现有单板播放器基础上叠加实现：P1 建立基础轮询，P2 支持动态指挥者切换，P3 处理演奏者静默故障，P4 处理指挥者静默故障。

**Meta 小组已商定的协议（来自讨论.md）：**

| 消息 ID | 名称 | 用途 |
|---------|------|------|
| 1 | `CONDUCTOR_CMD` | 子命令 play/stop/key/tempo |
| 2 | `DISCOVERY_PING` | 发现阶段广播 |
| 3 | `DISCOVERY_REPLY` | 回复自身 nodeId |
| 4 | `CONDUCTOR_ANNOUNCE` | 宣布指挥者 + 排序后的 nodeId 列表 |
| 5 | `TOKEN` | buf[0]=当前音符索引(0..31), buf[1]=failed nodeID(0 表示无故障) |
| 6 | `HEARTBEAT` | 当前演奏板/指挥者每 100ms 发送 |

- **Node ID** = 小组编号（固定），启动时由用户键盘输入 `node <id>` 配置，有效范围 1..14
- **Rank** = 在排序后的 `active_nodes` 列表中的位置（动态计算）
- **阶梯看门狗**：Rank0=200ms，Rank1=400ms，Rank2=600ms（每次收到 TOKEN 时重置）

## 第三步：Problem 1 — 轮询监听

### 3.1 辅助函数

```c
void add_active_node(App *self, int nodeId);

// 重新计算本板 rank（在 active_nodes 中的位置）

// 返回 active_nodes 中最小 rank 对应的 nodeId

```

### 3.2 网络发现（指挥者发起）

启动流程（`command_handler` 中 `play` 命令触发）：

```
do_discovery(App *self):
    add_active_node(self, node_id)
    add_known_node(self, node_id)
    discovery_done = 0
    广播 CAN_MSG_DISCOVERY_PING
    disc_session++
    SEND(MSEC(500), MSEC(10), self, discovery_timeout, disc_session)
```

`receiver()` 新增处理：

```
CAN_MSG_DISCOVERY_PING:
    // 演奏者收到 PING → 回复自身 nodeId
    发送 CAN_MSG_DISCOVERY_REPLY(buff[0]=self->node_id)

CAN_MSG_DISCOVERY_REPLY:
    // 指挥者收到回复 → 加入列表
    add_active_node(self, msg.buff[0])
    add_known_node(self, msg.buff[0])

CAN_MSG_CONDUCTOR_ANNOUNCE:
    // 所有板收到 → 重建网络状态
    conductor_id = msg.buff[0]
    重建 active_nodes（从 buff[1..N] 读取）
    recalculate_rank(self)
    discovery_done = 1
    if role == CONDUCTOR_ROLE && conductor_id == node_id:
        启动第一个 TOKEN(note_index=0)
```

`discovery_timeout(App *self, int session)`:

```
if session != disc_session: return
recalculate_rank(self)
广播 CAN_MSG_CONDUCTOR_ANNOUNCE(buff[0]=node_id, buff[1..N]=sorted active_nodes)
discovery_done = 1
广播 TOKEN(buff[0]=0, buff[1]=0)   // note_index=0, no failed node
```

### 3.3 令牌传递（替代原 play_note / stop_note 循环）

`receiver()` 处理 `CAN_MSG_TOKEN`：

```
last_token_index  = note_index（从 buff[0] 解码 0..31）
last_token_sender = msg.nodeId
reset_watchdog(self)        // 重置看门狗定时器

// 判断是否轮到本板
int player_rank = note_index % active_count
if my_rank == player_rank && !is_silent:
    play_one_note(self, note_index)
```

`play_one_note(App *self, int note_index)`:

```
1. heartbeat_session++ 然后立即启动 send_heartbeat 循环
2. 计算 period = get_period(self, brother_john_melodies[note_index % 32])
3. 计算 duration = beat_lengths[note_index % 32] * (30000 / tempo) ms
4. SYNC(&tone_task, tone_set_period, period)
5. if !muted: SYNC(&tone_task, tone_set_mute, 0)
6. SEND(MSEC(duration*0.9), MSEC(2), self, finish_note, pack(note_index, play_session))
   （提前 10% 结束音符，留 gap 给下一板）
```

`finish_note(App *self, int packed)`:

```
解包 note_index 和 session
if session != play_session: return
heartbeat_session++          // 停止心跳
SYNC(&tone_task, tone_set_mute, 1)
int next_index = (note_index + 1) % 32
广播 TOKEN(buff[0]=next_index, buff[1]=failed_node 或 0)
```

**指挥者控制命令：**

指挥者输入 `play`/`stop`/`key N`/`tempo N` 时：
- `play` → `do_discovery()` → 完成后自动发第一个 TOKEN
- `stop` → 广播 `CONDUCTOR_CMD(CAN_SUB_STOP)`，同时 `apply_stop(self)`
- `key/tempo` → 广播 `CONDUCTOR_CMD(CAN_SUB_KEY/TEMPO, value)` + 本地执行

演奏者收到 `CONDUCTOR_CMD`：
- `CAN_SUB_START`: `apply_play(self)`（但实际上演奏者等 TOKEN，不自行启动）
- `CAN_SUB_STOP`: `apply_stop(self)`；重置所有看门狗
- `CAN_SUB_KEY/TEMPO`: `apply_key()`/`apply_tempo()`

### 3.4 演奏者静音（'T' 键）

仅影响本地音频，不影响 TOKEN 传递：

```c
// 在 command_handler 中

    SYNC(&tone_task, tone_set_mute, self->mute);
```

## 第四步：Problem 2 — 拥抱你的新指挥

### 4.1 声称指挥权

**触发方式：**
- 控制台命令 `claim`（仅演奏者有效）
- 5 秒 PRESS-AND-HOLD USER 按钮（演奏者）

```c
void claim_conductorship(App *self):
    // 仅发送 buff[0]=nodeId，不含 nodeId 列表（区别于初始 ANNOUNCE）
    发送 CAN_MSG_CONDUCTOR_ANNOUNCE(buff[0]=node_id, length=1)
```

### 4.2 冲突解决与状态转换

`receiver()` 处理 `CAN_MSG_CONDUCTOR_ANNOUNCE`（`msg.length == 1` 时为声称型）：

```
int claimer = msg.buff[0]
int claimer_rank = rank_of(claimer, active_nodes)
int current_rank = rank_of(conductor_id, active_nodes)

// rank 越小优先级越高（对应 nodeId 越小）
if claimer_rank < current_rank || conductor_id == -1:
    conductor_id = claimer
    if claimer == node_id:
        // 本板成为新指挥者
        role = CONDUCTOR_ROLE
        print("Claimed Conductorship\n")
        LED: 以 tempo 频率闪烁（启动 led_blink 定时器）
    else if role == CONDUCTOR_ROLE:
        // 本板是旧指挥者，被取代
        role = MUSICIAN_ROLE
        print("Conductorship Void\n")
        LED: 常亮
```

### 4.3 USER 按钮处理（EXTI 中断）

```c
void button_press(App *self, int unused):   // 按下
    press_session++
    // 无论角色都启动 2s 计时
    SEND(SEC(2), SEC(1), self, button_2s, press_session)
    // 演奏者额外启动 5s 计时
    if role == MUSICIAN_ROLE:
        SEND(SEC(5), SEC(1), self, button_5s, press_session)

void button_release(App *self, int unused): // 松开
    press_session++   // 失效所有挂起的计时器

void button_2s(App *self, int session):
    if session != press_session: return
    if role == CONDUCTOR_ROLE:
        apply_key(self, 0)
        apply_tempo(self, 120)
        print("Key and tempo reset to defaults\n")
    elif role == MUSICIAN_ROLE:
        // 默认进入 F2（若用户输入 'Z' 则锁定 F1）
        enter_silent_failure(self, SILENT_F2)

void button_5s(App *self, int session):
    if session != press_session: return
    if role == MUSICIAN_ROLE:
        claim_conductorship(self)
```

## 第五步：Problem 3 — 沉默之声（演奏者静默故障）

### 5.1 静默故障进入 / 退出

**命令：**
- `f1` → 切换 F1 模式（进入/退出）
- `f2` → 进入 F2 模式（自动恢复）
- F3 → 自动检测（CAN_SEND 失败或 F3 检测逻辑）

```c
void enter_silent_failure(App *self, SilentMode mode):
    is_silent = 1
    silent_mode = mode
    silent_session++
    heartbeat_session++       // 停止心跳
    watchdog_session++        // 停止看门狗（自身不触发接管）
    cond_wd_session++         // 停止指挥者看门狗
    apply_stop(self)          // 停止音频输出
    print("Silent Failure\n")
    LED: 关闭（unlit）
    // F2：启动自动退出定时器
    if mode == SILENT_F2:
        int delay_ms = 5000 + (pseudo_rand() % 5001)  // [5000, 10000] ms
        SEND(MSEC(delay_ms), MSEC(100), self, f2_auto_exit, silent_session)

void f2_auto_exit(App *self, int session):
    if session != silent_session: return
    leave_silent_failure(self)

void leave_silent_failure(App *self):
    is_silent = 0
    silent_mode = SILENT_NONE
    print("Leave Silent Failure\n")
    // 广播 DISCOVERY_REPLY 重新加入
    send_can(CAN_MSG_DISCOVERY_REPLY, node_id, ...)
    // 重新加入后等待 TOKEN 自动同步，不主动播放
    reset_watchdog(self)
    start_print_timer(self)
    LED: 绿色亮起
    // P4：若旧指挥者恢复，以演奏者身份加入
    if was_conductor:
        role = MUSICIAN_ROLE
        print("I Now Join As A Musician\n")
```

**F3 检测：**
- 每次 `CAN_SEND` 返回错误 → `enter_silent_failure(self, SILENT_F3)`
- `receiver()` 收到任何 CAN 消息时，若 `silent_mode == SILENT_F3` → `leave_silent_failure()`

**F1 的 'Z' 键锁定：**

```c
// 在 command_handler 中
case 'z':
    if is_silent && silent_mode == SILENT_F2:
        silent_session++    // 取消 F2 自动退出定时器
        silent_mode = SILENT_F1
        print("Locked to F1 mode\n")
    elif is_silent && silent_mode == SILENT_F1:
        leave_silent_failure(self)
```

### 5.2 心跳发送

```c
void send_heartbeat(App *self, int session):
    if session != heartbeat_session: return
    if is_silent: return
    send_can(CAN_MSG_HEARTBEAT, node_id, ...)
    SEND(MSEC(10), MSEC(2), self, send_heartbeat, session)
```

在 `play_one_note()` 开始时启动；在 `finish_note()` 中通过 `heartbeat_session++` 停止。

### 5.3 阶梯看门狗

```c
void reset_watchdog(App *self):
    watchdog_session++
    Time delay = MSEC(200 + self->rank * 200)
    // rank0=200ms, rank1=400ms, rank2=600ms
    SEND(delay, MSEC(10), self, watchdog_fire, watchdog_session)

void watchdog_fire(App *self, int session):
    if session != watchdog_session: return
    if is_silent: return
    // 识别失败的板：last_token_sender 应播放但没有后续 TOKEN
    int failed_node = last_token_sender
    // 或：根据 last_token_index % active_count 推断应播放的 rank
    int failed_rank = last_token_index % active_count
    int failed_nodeId = active_nodes[failed_rank]
    remove_active_node(self, failed_nodeId)
    recalculate_rank(self)
    // 接管：播放该音符
    play_one_note(self, last_token_index)
```

### 5.4 成员状态显示（'M' 命令，仅指挥者）

```c
// command_handler 中 "membership" 或 "M" 命令
for i in 0..known_count:
    nodeId = known_nodes[i]
    status = is_in_active(nodeId) ? "active" : "silent_failure"
    printf("Board %d: %s\n", nodeId, status)
```

### 5.5 恢复板重新同步

收到 `CAN_MSG_DISCOVERY_REPLY`（其他板恢复时广播）：

```c
add_active_node(self, msg.buff[0])
recalculate_rank(self)
```

恢复板下次收到 TOKEN 时，若 `note_index % active_count == my_rank`，自动开始演奏 → 实现 ≤1s 的自动同步。

## 第六步：Problem 4 — 让指挥者静默

### 6.1 指挥者心跳（指挥者不演奏时也发送）

```c
void conductor_heartbeat(App *self, int session):
    if session != heartbeat_session: return
    if is_silent: return
    if role != CONDUCTOR_ROLE: return
    send_can(CAN_MSG_HEARTBEAT, node_id, ...)

```

在成为指挥者时启动；演奏者的心跳由 `send_heartbeat`（每 100ms）覆盖。

### 6.2 指挥者看门狗（所有板运行，与音符看门狗独立）

```c
void reset_conductor_watchdog(App *self):
    cond_wd_session++
    Time delay = MSEC(200 + self->rank * 200)
    SEND(delay, MSEC(10), self, conductor_wd_fire, cond_wd_session)

void conductor_wd_fire(App *self, int session):
    if session != cond_wd_session: return
    if is_silent: return
    // 指挥者失联
    remove_active_node(self, conductor_id)
    recalculate_rank(self)
    // rank 最低的存活板成为新指挥者
    if node_id == lowest_active_node(self):
        become_new_conductor(self)
```

`receiver()` 中：每次收到来自指挥者的 HEARTBEAT → `reset_conductor_watchdog(self)`。

### 6.3 自动选举

```c
void become_new_conductor(App *self):
    role = CONDUCTOR_ROLE
    conductor_id = node_id
    print("I Am The New Conductor\n")
    LED: 以 tempo 频率闪烁
    // 广播含完整 nodeId 列表的 CONDUCTOR_ANNOUNCE
    broadcast_conductor_announce(self)
    // 启动指挥者心跳
    heartbeat_session++
    SEND(MSEC(100), MSEC(10), self, conductor_heartbeat, heartbeat_session)
    // 继续旋律（从 last_token_index 继续）
    reset_watchdog(self)
    // 如果 active_count >= 1，发一个 TOKEN 恢复旋律
    if active_count > 0:
        广播 TOKEN(last_token_index)
```
